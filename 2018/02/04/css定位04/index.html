CSS属性--定位

发表于 2018-02-04 | 分类于 CSS3
字数统计: 2,772 | 阅读时长 ≈ 10

CSS定位属性

position

取值：
static：默认值。没有定位，元素出现在正常的流中（忽略 top, bottom, left, right 或者 z-index 声明）
relative：生成相对定位的元素，相对于其正常位置进行定位。”left:20” 会向元素的 LEFT 位置添加 20 像素。同理可得right、top和bottom的位置偏移情况。
absolute：生成绝对定位的元素，相对于 static 定位以外的第一个父元素进行定位。元素的位置通过 “left”, “top”, “right” 以及 “bottom” 属性进行规定。
fixed：生成绝对定位的元素，相对于浏览器窗口进行定位。元素的位置通过 “left”, “top”, “right” 以及 “bottom” 属性进行规定。当出现滚动条时，对象不会随着滚动。
sticky：它就像是relative和fixed的合体，当在屏幕中时按常规流排版，当卷动到屏幕外时则表现如fixed。

left

该属性定义了定位元素左外边距边界与其包含块左边界之间的偏移。
取值：
auto：默认值，通过浏览器计算左边缘的位置。
%：设置以包含元素的百分比计的左边位置。可使用负值。
length：用 px、cm 等单位设置元素的左边位置。可使用负值。
计算值说明：position为relative时，如果right和left都是auto，则它们的计算值是0；如果right和left其中一个为auto，则auto相当于另一个的负值，即left = -right；如果right和left的值都不为auto，则忽略right。

right

该属性定义了定位元素右外边距边界与其包含块右边界之间的偏移。
auto：默认值。通过浏览器计算右边缘的位置。
%：设置以包含元素的百分比计的右边位置。可使用负值。
length：使用 px、cm 等单位设置元素的右边位置。可使用负值。

top

该属性定义了一个定位元素的上外边距边界与其包含块上边界之间的偏移。
auto：默认值。通过浏览器计算上边缘的位置。
%：设置以包含元素的百分比计的上边位置。可使用负值。
length：使用 px、cm 等单位设置元素的上边位置。可使用负值。
计算值说明：position为relative时，如果top和bottom都是auto，则它们的计算值是0；如果top和bottom其中一个为auto，则auto相当于另一个的负值，即top = -bottom；如果top和bottom的值都不为auto，则忽略bottom。

bottom

该属性定义了定位元素下外边距边界与其包含块下边界之间的偏移。
auto：默认值。通过浏览器计算底部的位置。
%：设置以包含元素的百分比计的底边位置。可使用负值。
length：使用 px、cm 等单位设置元素的底边位置。可使用负值。

z-index

设置元素的堆叠顺序。拥有更高堆叠顺序的元素总是会处于堆叠顺序较低的元素的前面。可以为负值，适用于非static定位的元素。

clip

clip 这个属性用于定义一个剪裁矩形。对于一个绝对定位元素(只有设置position为absolute会fixed属性的元素才会生效)，在这个矩形内的内容才可见。出了这个剪裁区域的内容会根据 overflow 的值来处理。剪裁区域可能比元素的内容区大，也可能比内容区小。
取值：
shape：设置元素的形状。唯一合法的形状值是：rect (top, right, bottom, left)，依据上-右-下-左的顺序提供自对象左上角为(0,0)坐标计算的四个偏移数值，其中任一数值都可用auto替换，即此边不剪切。
相当于在x等于top和bottom的值处画两条水平线，在y等于left和right的值处画两条垂直线，然后四条线围成的区域就是可以显示出来的区域。
auto：默认值。不应用任何剪裁。

display

取值：
none：隐藏对象。与visibility属性的hidden值不同，其不为被隐藏的对象保留其物理空间。
inline：指定对象为内联元素。
block：指定对象为块元素。
inline-block：指定对象为内联块元素。
list-item：指定对象为列表项目。
table：指定对象为块元素级的表格，类似于table标签，表格前后带有换行符
inline-table：指定对象为内联元素级的表格，类似于table标签，表格前后没有换行符
table-caption：指定对象为表格标题，类似于caption标签
table-cell：指定对象作为表格单元格，类似于td标签
table-row：指定对象作为表格的行，类似于tr标签
table-row-group：指定对象作为表格行组，类似于tbody标签
table-column：指定对象作为表格列，类似于col标签
table-column-group：指定对象作为表格列祖，类似于colgroup标签
table-header-group：指定对象作为表格标题组，类似于thead标签
table-footer-group：指定对象作为表格脚注组，类似于tfoot标签
flex，inline-flex：弹性伸缩盒
run-in：根据上下文决定作为块级元素还是内联元素显示

visibility

定义了元素是否可见。与display属性不同，visibility会为隐藏的元素保留其占据的物理空间
取值：
visible：默认值，元素是可见的
hidden：元素是不可见的
collapse：当在表格元素中使用时，此值可删除一行或一列，但是它不会影响表格的布局。被行或列占据的空间会留给其他内容使用。如果此值被用在其他的元素上，会呈现为 “hidden”。
注意：表格元素使用hidden，页面不会呈现对应的表格元素，但是会占据空间；表格元素使用collapse，页面不会呈现表格元素，也不会占据空间，可以利用此特性来删除表格的行或列。

vertical-align

设置元素的垂直对齐方式。适用于行内元素和table-cell元素。
该属性定义行内元素的基线相对于该元素所在行的基线的垂直对齐。允许指定负长度值和百分比值。在单元格中，这个属性会设置单元格框中的单元格内容的对齐方式。
（对于行内元素而言，都会有一条基线，这条基线就是在其中放置一个x字母时，x字母最下边所占据的一条线。）
取值：
baseline，默认。把当前盒的基线与父级盒的基线对齐，如果该盒没有基线，就将底部外边距的边界和父级的基线对齐。
sub，把当前盒的基线降低到合适的位置作为父级盒的下标
super，把当前盒的基线提升到合适的位置作为父级盒的上标
top，把当前盒的top和行盒的top对齐
text-top，把当前盒的top和父级的内容区的top对齐
middle，把当前盒的垂直中心和父级盒的基线加上父级的搬半x-height对齐(其实就是x字母的中间位置)
bottom，把当前盒的bottom和行盒的bottom对齐
text-bottom，把当前盒的bottom和父级的内容区的bottom对齐
length，把当前盒提升或降低这个距离。当值为0时相当于baseline。
%，使用 “line-height” 属性的百分比值来排列此元素，把当前盒提升或降低这个距离。当值为0%时等同于baseline。

补充：解决下面这种情况下，图片底部的空隙情况。（在HTML5文档声明下，块状元素内部的内联元素的行为表现，就好像块状元素内部还有一个（更有可能两个-前后）看不见摸不着没有宽度没有实体的空白节点，这个假想又似乎存在的空白节点，称之为“幽灵空白节点”–张鑫旭。）

解决方案：

  • img标签设置vertical-align：bottom|top|middle|text-top|text-bottom|sub，不要是super和baseline即可
  • img标签设置display：block
  • p标签设置font-size：0或者line-height：0

    float

    定义了元素向左或者向右浮动放置。
    取值：
    left：元素向左浮动
    right：元素向右浮动
    none：默认值
    假如在一行之上只有极少的空间可供浮动元素，那么这个元素会跳至下一行，这个过程会持续到某一行拥有足够的空间为止。
    当一个元素是绝对定位元素或者定义了display为none时，float定义不生效。

    clear

    clear 属性规定元素的哪一侧不允许其他浮动元素。
    取值：
    none：允许两边都可以有浮动对象
    both：不允许有浮动对象
    left：不允许左边有浮动对象
    right：不允许右边有浮动对象

    cursor

    cursor 属性定义了鼠标指针放在一个元素边界范围内时所用的光标形状。
    取值：
    default：默认光标（通常是一个箭头）
    auto：默认。浏览器设置的光标
    crosshair：光标呈现为十字线
    pointer：光标呈现为指示链接的指针（一只手）
    move：此光标指示某对象可被移动
    e-resize：此光标指示矩形框的边缘可被向右（东）移动
    ne-resize：此光标指示矩形框的边缘可被向上及向右移动（北/东）
    nw-resize：此光标指示矩形框的边缘可被向上及向左移动（北/西）
    n-resize：此光标指示矩形框的边缘可被向上（北）移动
    se-resize：此光标指示矩形框的边缘可被向下及向右移动（南/东）
    sw-resize：此光标指示矩形框的边缘可被向下及向左移动（南/西）
    s-resize：此光标指示矩形框的边缘可被向下移动（南）
    w-resize：此光标指示矩形框的边缘可被向左移动（西）
    text：此光标指示文本
    wait：此光标指示程序正忙（通常是一只表或沙漏）
    help：此光标指示可用的帮助（通常是一个问号或一个气球）

    overflow

    规定当内容溢出元素框时发生的事情。
    visible，默认值。内容不会被修剪，会呈现在元素框之外。
    hidden，内容会被修剪，并且其余内容是不可见的。
    scroll，内容会被修剪，但是浏览器会显示滚动条以便查看其余的内容。
    auto，如果内容被修剪，则浏览器会显示滚动条以便查看其余的内容。
    clip：和hidden一样，clip也被用来隐藏溢出容器的内容且不出现滚动条，但是clip是一个完全禁止滚动的容器，而hidden仍然可以通过编程机制让内容可以滚动，浏览器目前不支持。
